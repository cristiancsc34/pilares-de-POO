Polimorfismo em POO

O que é Polimorfismo?

Polimorfismo é um dos pilares da Programação Orientada a Objetos (POO) que permite que objetos de diferentes classes sejam tratados de maneira uniforme. Em outras palavras, o polimorfismo permite que diferentes classes possuam métodos com o mesmo nome, mas com comportamentos distintos. Isso aumenta a flexibilidade e a reutilização de código.

O polimorfismo pode ser alcançado de duas formas: sobrecarga e sobrescrita de métodos. A sobrecarga permite que um método tenha múltiplos comportamentos baseados nos parâmetros, enquanto a sobrescrita de métodos permite que uma classe filha substitua um método da classe pai com uma implementação específica.

Exemplo de Código de Polimorfismo em POO

class Animal:
    def falar(self):
        raise NotImplementedError("Subclasses devem implementar esse método")

class Cachorro(Animal):
    def falar(self):
        return "Au Au!"

class Gato(Animal):
    def falar(self):
        return "Miau!"

def fazer_animal_falar(animal: Animal):
    print(animal.falar())

# Criando objetos das classes filhas
cachorro = Cachorro()
gato = Gato()

# Chamando o método falar das classes filhas
fazer_animal_falar(cachorro)
fazer_animal_falar(gato)

Explicação do Código

Classe Animal:
A classe `Animal` é a classe base, que possui o método `falar`. Porém, esse método é definido como abstrato, utilizando a exceção `NotImplementedError`, indicando que ele precisa ser implementado pelas classes filhas.

Classes Cachorro e Gato:
As classes `Cachorro` e `Gato` herdam de `Animal` e implementam o método `falar` de maneira específica para cada tipo de animal. No caso do cachorro, ele retorna "Au Au!", enquanto o gato retorna "Miau!". Cada classe tem seu comportamento particular.

Função fazer_animal_falar:
A função `fazer_animal_falar` recebe um objeto do tipo `Animal` e chama o método `falar`. Graças ao polimorfismo, a função pode tratar tanto o `Cachorro` quanto o `Gato` de maneira uniforme, já que ambos têm o método `falar`. Mesmo que as implementações sejam diferentes, o código que chama o método não precisa saber exatamente que tipo de animal é. Isso demonstra o poder do polimorfismo em tornar o código mais flexível e reutilizável.

Benefícios do Polimorfismo:
O polimorfismo permite que diferentes classes implementem comportamentos específicos para um mesmo método, facilitando a manutenção e a expansão do código. Em sistemas grandes e complexos, isso pode melhorar a organização e a legibilidade do código, além de tornar a interação entre objetos mais simples e intuitiva.
