Encapsulamento em POO

O que é Encapsulamento?

Encapsulamento é um dos pilares da Programação Orientada a Objetos (POO). Ele envolve esconder os detalhes internos do funcionamento de um objeto, permitindo que somente as operações permitidas sejam acessadas externamente, e os dados sejam protegidos contra modificações não autorizadas. Isso é feito através da utilização de modificadores de acesso como `private`, `protected`, e `public` nas classes.

O encapsulamento melhora a segurança do sistema e permite que os objetos se comportem de maneira previsível e controlada. Isso ajuda na manutenção e organização do código, pois o comportamento interno de um objeto pode ser alterado sem impactar o resto do sistema, desde que a interface pública permaneça a mesma.

Exemplo de Código de Encapsulamento em POO

class ContaBancaria:
    def __init__(self, saldo: float):
        self.__saldo = saldo  # Atributo privado, não acessível diretamente de fora da classe

    def depositar(self, valor: float):
        if valor > 0:
            self.__saldo += valor
            print(f"Depósito de R$ {valor:.2f} realizado.")
        else:
            print("Valor inválido para depósito!")

    def retirar(self, valor: float):
        if 0 < valor <= self.__saldo:
            self.__saldo -= valor
            print(f"Retirada de R$ {valor:.2f} realizada.")
        else:
            print("Saldo insuficiente ou valor inválido!")

    def obter_saldo(self):
        return self.__saldo  # Método público que permite acesso ao saldo

# Criando uma conta bancária
conta = ContaBancaria(1000.00)

# Realizando operações
conta.depositar(500.00)
conta.retirar(200.00)
print(f"Saldo atual: R$ {conta.obter_saldo():.2f}")

Explicação do Código

Classe ContaBancaria:
A classe `ContaBancaria` tem um atributo privado chamado `__saldo`, que não pode ser acessado diretamente de fora da classe, garantindo que o valor do saldo seja protegido.

Modificadores de Acesso:
O encapsulamento é implementado através da utilização do modificador de acesso `private` (indicando um atributo privado) e `public` para os métodos, como o `depositar`, `retirar` e `obter_saldo`, que permitem interagir com o objeto de maneira controlada. O acesso direto ao saldo é bloqueado para que o valor só possa ser alterado ou consultado de maneira segura.

Benefícios do Encapsulamento:
Ao proteger o atributo `__saldo`, o código fica mais seguro, pois não é possível alterar diretamente o valor sem passar pelas verificações dos métodos. Além disso, o encapsulamento torna o código mais organizado e flexível, permitindo que a lógica interna da classe seja modificada sem afetar as outras partes do sistema que interagem com ela.
